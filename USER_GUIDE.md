# MavixBoard — Руководство по установке и запуску борта

Документ составлен по ГОСТ 19.505-79 (для бортового ПО; установку и сопровождение
ведёт инженер, эксплуатацию полёта — оператор через MavixDesktop).

## 1. Аннотация
Руководство по подготовке и запуску борта MavixBoard на дроне (Raspberry Pi)
в составе системы доставки грузов Mavix.

## 2. Назначение программы
Борт передаёт видео и телеметрию оператору, исполняет команды управления и сброс
груза, проброшенные с приложения оператора через полётный контроллер.

## 3. Условия выполнения программы
- Raspberry Pi с Ubuntu, камеры (USB/CSI), полётный контроллер по UART/USB.
- Интернет на борту (Wi-Fi/LTE), доступ к серверу Mavix.
- Подготовленное железо и права на порты — [HARDWARE_SETUP.md](HARDWARE_SETUP.md).

## 4. Выполнение программы

### 4.1 Подготовка (однократно)
1. ОС, сеть/LTE, камеры, UART и **права на serial-порты** — по HARDWARE_SETUP §1.
2. Прошивка и настройка полётного контроллера — HARDWARE_SETUP §2.
3. Системные GStreamer-зависимости и Python-пакеты — HARDWARE_SETUP §1.6.

### 4.2 Регистрация и запуск
1. В кабинете администратора получить `ADMIN_ID` и `ENROLLMENT_TOKEN`; задать их в env.
2. Запустить борт:
   ```bash
   python -m mavixboard
   ```
   При первом запуске борт регистрируется (enrollment), получает `DRONE_TOKEN`/
   имя и дописывает их в env-файл; далее использует сохранённую идентичность.
3. Автозапуск — systemd-юнит (HARDWARE_SETUP §1.8), группы `dialout video`.

### 4.3 Эксплуатация в полёте
Оператор принимает заявку и подключается к дрону из MavixDesktop; видео,
телеметрия и команды идут по WebRTC напрямую. Сброс груза — кнопкой джойстика
(AUX) или на экране управления.

## 5. Сообщения оператору/инженеру
- Борт не появляется у оператора → проверить сеть, регистрацию (enrollment) и
  сигналинг; журнал борта (`journalctl -u mavix`).
- Нет видео / ICE failed → STUN/TURN и совместимость — [WEBRTC_TURN_NOTES.md](WEBRTC_TURN_NOTES.md).
- FC не армится / RXLOSS → диагностика в [HARDWARE_SETUP.md](HARDWARE_SETUP.md) §3.
