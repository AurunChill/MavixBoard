# MavixBoard

MavixBoard — дашборд-приложение Mavix.

## Генерация токенов

Токены генерируются с помощью `secrets.token_hex()` из стандартной библиотеки Python.

### Цепочка вызовов

```
secrets.token_hex()
    └── os.urandom(n)             # Python stdlib
            ├── getrandom()       # Linux (syscall ядра, начиная с Linux 3.17)
            │   или /dev/urandom  # запасной вариант для старых ядер
            └── BCryptGenRandom() # Windows (bcrypt.dll)
```

Python здесь — тонкая обёртка. Вся работа выполняется ядром ОС, которое собирает энтропию из физических источников: джиттер прерываний, аппаратные события, сетевые пакеты, аппаратный RNG (RDRAND на x86).

**Стек: Python → ядро ОС → железо.**

### Вероятность коллизии

Каждый символ токена — шестнадцатеричная цифра (`0–9`, `a–f`), то есть 16 вариантов. Для токена длиной 64 символа пространство значений:

```
N = 16^64 = 2^256 ≈ 1.16 × 10^77
```

Это то же пространство, что у SHA-256 — больше атомов в наблюдаемой вселенной.

Вероятность коллизии считается через задачу о днях рождения. Если сгенерировать `k` токенов, вероятность что хотя бы два совпадут:

```
P ≈ k² / 2N
```

**Пример** — `L=64`, `k=1 000 000 000` (миллиард токенов):

```
k² = 10^18
2N = 2 × 1.16 × 10^77 ≈ 2.32 × 10^77

P ≈ 10^18 / 2.32 × 10^77 ≈ 4.3 × 10^-60
```

Для сравнения — вероятность выпасть орлом 200 раз подряд:

```
P(орёл × 200) = (1/2)^200 = 2^(-200) ≈ 10^(-60)
```

Коллизия миллиарда токенов (`4.3 × 10^-60`) примерно равна по вероятности этому событию.

## Хранение токена

Токен сохраняется в файл без сторонних зависимостей.

**Путь:** `~/.config/mavixboard/token`

Директория создаётся автоматически при первой записи. Права доступа к файлу рекомендуется ограничить вручную:

```bash
chmod 600 ~/.config/mavixboard/token
```

### API

```python
from mavixboard.token import storage

storage.write('your_token')  # записать токен на диск
token = storage.get()        # прочитать токен ('' если файла нет)
```

## Регистрация дрона

При каждом запуске программа отправляет запрос на регистрацию дрона на сервер.

**Endpoint:** `POST /api/v1/drones/register`

**Тело запроса:**

```json
{
  "user_id": "<USER_ID из .env>",
  "drone_id": "<64-символьный hex токен дрона>"
}
```

**Ожидаемый ответ (201):**

```json
{
  "drone_id": "...",
  "user_id": "...",
  "drone_token": "..."
}
```

Регистрация считается успешной только при статусе `201` и наличии всех трёх полей в ответе. В остальных случаях логируется предупреждение.

## Трансляция видео (GStreamer)

Видеопоток с USB-камер передаётся по WebRTC через GStreamer.

> **Поддерживаются только USB-камеры (V4L2).** CSI-камеры (например, Camera Module для Raspberry Pi) не поддерживаются.

### Требования

```bash
sudo apt install gstreamer1.0-tools gstreamer1.0-plugins-base \
    gstreamer1.0-plugins-good gstreamer1.0-plugins-bad \
    gstreamer1.0-plugins-ugly gstreamer1.0-libav \
    python3-gi gir1.2-gst-plugins-base-1.0 v4l-utils
```

### Архитектура

| Компонент | Описание |
|---|---|
| `V4l2Scanner` | Сканирует устройства через `v4l2-ctl`, определяет поддерживаемые форматы и разрешения |
| `CameraCalibrator` | Проверяет каждый режим камеры запуском GStreamer-пайплайна — исключает нерабочие режимы |
| `CameraRegistry` | Кэширует список камер, сохраняет конфигурацию в `_data/<name>.json` |
| `PipelineBuilder` | Строит строку GStreamer-пайплайна для WebRTC-трансляции |
| `GStreamerPipe` | Управляет пайплайном: запуск, остановка, динамическое изменение битрейта |

### Поддерживаемые форматы

- `YUYV` (YUY2) — сырой формат без сжатия
- `MJPG` (Motion JPEG) — аппаратное сжатие камеры

Все форматы перекодируются в H.264 (`x264enc`) и передаются по RTP внутри WebRTC.

### WebRTC: STUN и TURN

STUN и TURN серверы задаются в `.env`:

```
STUN_SERVER=stun://your-server:3478
TURN_SERVER=turn://user:password@your-server:3478
```

TURN опционален — если не задан, используется только STUN. Оба значения подставляются в WebRTC-пайплайн автоматически.

## Дистрибуция

MavixBoard распространяется через сайт Mavix. Токен пользователя (`USER_ID`) вшивается в дистрибутив автоматически при скачивании.

### Механизм

1. Пользователь регистрируется на сайте — сервер генерирует уникальный `USER_ID` (64-символьный hex токен)
2. При нажатии "Скачать" сервер формирует `.zip` архив в памяти:
   - `mavixboard-x.x.x-py3-none-any.whl` — собранный пакет со всем кодом
   - `.env` — с `USER_ID` пользователя уже внутри
3. Пользователь скачивает архив и запускает программу — авторизация происходит автоматически

### Сборка пакета

```bash
python -m build
```

Артефакт появится в `dist/mavixboard-x.x.x-py3-none-any.whl`. Этот файл кладётся на сервер один раз и переиспользуется для всех пользователей.
